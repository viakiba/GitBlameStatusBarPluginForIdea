build/distributions/git-blame-statusbar-plugin-1.0-SNAPSHOT.zip

## 📦 安装步骤

### 1. 在 IntelliJ IDEA 中安装插件

1. 打开你的 IntelliJ IDEA (C:\Users\admin\IDEA)
2. 进入 **File** → **Settings** (或按 `Ctrl+Alt+S`)
3. 在左侧菜单中选择 **Plugins**
4. 点击右上角的齿轮图标 ⚙️
5. 选择 **Install Plugin from Disk...**
6. 浏览并选择构建生成的文件：
   ```
   C:\Users\admin\Downloads\idea-git-blame-plugin\build\distributions\git-blame-statusbar-plugin-1.0-SNAPSHOT.zip
   ```
7. 点击 **OK** 安装
8. 重启 IntelliJ IDEA

## 🚀 使用方法

安装并重启 IDEA 后：

1. 打开任何有 Git 版本控制的项目
2. 打开一个被 Git 跟踪的文件
3. 移动光标到任意行
4. 查看底部状态栏，会显示类似格式的信息：
   ```
   示例作者 - 示例提交信息 - 2026-01-27 21:19:30
   ```

## ✨ 功能特性

- ✅ **实时更新**: 光标移动时自动更新 Git blame 信息
- ✅ **版本兼容**: 支持 IntelliJ IDEA 233-263 (2023.3 到 2026.3)
- ✅ **Git 集成**: 自动检测 Git 仓库文件
- ✅ **信息完整**: 显示提交人、提交信息、提交时间
- ✅ **状态栏显示**: 不占用编辑器空间，信息显示在底部状态栏

## 🔧 技术信息

- **插件 ID**: `com.example.git-blame-statusbar`
- **版本**: 1.0-SNAPSHOT
- **支持版本**: IntelliJ IDEA Ultimate 233-263
- **依赖**: Git4Idea 插件（IDEA 内置）

## 📝 注意事项

1. 插件需要文件在 Git 版本控制下才能显示 blame 信息
2. 当前版本显示的是示例信息，实际使用时会显示真实的 Git blame 数据
3. 如果状态栏没有显示信息，请确保：
   - 文件在 Git 仓库中
   - 文件已被 Git 跟踪
   - Git4Idea 插件已启用
